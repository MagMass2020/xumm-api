We're happy to announce this new release of XUMM to 0.5.1, containing well over 50 improvements and fixes. Although the version number indicates this is to be a minor release, the effort and changes are more like a major release. There are some important changes and fixes added to this release after we already communicated the version number.

We have listened to your feedback and we will continue to do so. We appreciate your effort for making XUMM a better app everyday. Special thanks go out to @NixerFFM (Twitter), @saschofield52 (Twitter) & Towo Labs (xrptoolkit.com).

Added / Improved

  • The transaction detail screen got a make over (if you tap a transaction on the Events tab)
  • Added Support for "Checks" (new XRP ledger transaction type) & "DepositPreauth" transaction.
  • Minor user interface updates to improve usability
  • TrustLines with negative balances (self issued IOU's) now show in the XUMM home screen (obligation) and are available for sending (issuing)
  • Show local date & time for transaction details page and show account info like this, and show year in events page if it is not the current year
  • Allow hiding your account address and balance by default, remember state for next app start
  • Improved exchange rates & liquidity checking for sending/exchanging assets (based on the value requested)
  • Show received value for currency exchanges & the currency code in events page
  • Show received XRP in Events list with an AccountDelete transaction
  • Improved secret numbers validation on generating new XRP account
  • Allow signing with all accounts for "sign in requests" (eg. unactivated accounts)
  • Tapping the XUMM app version (Settings, Advanced) will popup the release notes
  • Improved "maximum amount reached"-message for sending IOU currencies, updating the value to the max. spendable amount
  • Hide balance section on Account home screen if the account is not activated
  • Explanation for unactivated accounts regarding the 20 XRP reserve (when importing/adding an account)
  • Improved user flow (and on screen information) when importing rekeyed accounts like Toast vanity addresses (using "Regular keys")

Changed / Fixed

  • Improved XRP ledger (node) connectivity
  • Backspace first PIN digit didn't work on all devices
  • Sign button on request not reachable on iOS
  • Numpad fix for iOS
  • Fixed blank screen on submitting signed transaction by QR code
  • Scrolling stopped within switching an account after certain number
  • Support for 21:9 aspect ratio phones regarding UI behaviour
  • Regular key account showed as an read only account, changed to Read/Write
  • Account reserve list & balance calculation inaccuracy solved
  • On Android when clicking multiple times on share link the app will no longer open multiple share dialogs
